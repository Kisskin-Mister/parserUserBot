# IT CURATOR SKILL (Codex)

> **CRITICAL:** перед любыми действиями по новостям сначала прочитай этот файл целиком (`read_file`)
> работа только по прямому запросу пользователя
> в авто-режиме новости не трогаешь

## Режим и цель
- разбирать unread новости из базы
- делать deep-дайджест в стиле современных telegram-каналов
- релевантные новости пересылать в отдельный чат `-1002801950994`
- после пересылки давать ссылки на посты из этого чата в дайджесте
- чистить базу от мусора и дублей

## Важное разделение чатов
- `LOG_CHAT_ID` - только для вакансий (не менять)
- `NEWS_CHAT_ID` - только для релевантных новостей дайджеста

## Источник и инструменты
- unread: `python3 mcp_server.py news`
- сделать репост новости в news-чат: `python3 mcp_server.py repost_news <news_id>`
- удалить нерелевантную новость: `python3 mcp_server.py delete_news <news_id>`
- отметить как прочитанное: `python3 mcp_server.py mark_news <id1> <id2> ...`

## Алгоритм
1. получить unread через `news`
2. классифицировать каждую запись:
   - relevant: it, ai, devtools, github tools, релизы, важные апдейты
   - trash: дубли, оффтоп, флуд, слабая ценность
3. trash удалять сразу через `delete_news`
4. каждую relevant новость репостить как новое сообщение в `NEWS_CHAT_ID` через `repost_news`
5. собрать deep-дайджест по категориям:
   - 🤖 AI
   - 🛠 DevTools
   - 🐙 GitHub / Open Source
   - ☁️ Backend / Infra / Data
   - ⚡ Другое важное
6. в каждом пункте дать:
   - заголовок
   - 1-2 строки сути
   - эмодзи-маркер релевантности
   - ссылку на пересланный пост из news-чата
7. отметить все новости из дайджеста как прочитанные через `mark_news`

## Формат подачи
- как современный tg-дайджест
- с заглавной буквы
- аккуратные абзацы
- эмодзи по смыслу
- без сухого канцелярита

## Если релевантных новостей нет
- не спамить в news-чат
- пользователю дать короткий факт-отчет
